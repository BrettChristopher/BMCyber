NBA Active Directory IAM Project

A full Identity & Access Management (IAM) simulation using Microsoft Active Directory to model identity governance for an NBA organization. This project demonstrates RBAC, GPO enforcement, privileged access workflows, lifecycle automation, and auditing in a realistic enterprise-style environment.

Environment Overview

  • Domain: nba.local
  • Domain Controller: DC01
  • File Server: FS01
  • Application Server: APP01
  • OS: Windows Server 2022

Active Directory Architecture

The directory is structured around NBA organizational roles to support clean identity lifecycle management and least‑privilege access.

  • Players
  • Coaches
  • Analytics
  • Medical
  • FrontOffice
  • IT
  • PrivilegedRoles
  • DisabledUsers

RBAC & Access Controls

All access is assigned through AD security groups—never directly to users.

  • GRP_Players: Player portal, schedules
  • GRP_Coaches: Playbooks, scouting reports
  • GRP_Analytics: Dashboards, data models
  • GRP_Medical: PHI, injury reports
  • GRP_FrontOffice: Contracts, salary tools
  • GRP_ITAdmins: Admin-level privileges

File Server Permissions

NTFS permissions enforce separation of duties and protect sensitive team data.

  • Coaches → Modify /Coaches
  • Analytics → Modify /Analytics
  • Medical → Full Control /Medical
  • Players → Read /Players
  • Front Office → Modify /FrontOffice
  • All Staff → Read /Shared

Group Policy Enforcement

Security baselines and role-specific GPOs were deployed across the domain.

  • 14‑character minimum passwords
  • 24‑password history
  • 90‑day rotation
  • MFA for privileged accounts
  • BitLocker enforcement
  • Block unsigned scripts
  • Disable removable storage
  • SMB signing required

Privileged Access Management

A tiered admin model and Just‑In‑Time (JIT) elevation workflow were implemented to reduce standing privileges.

  • Tier 0: Domain Admins
  • Tier 1: Server Admins
  • Tier 2: Helpdesk
  • Temporary elevation into GRP_ITAdmins with approval

Identity Lifecycle Automation

Joiner–Mover–Leaver processes were automated using PowerShell.

  • Joiner: Auto‑create account, assign groups, provision home folder
  • Mover: Update group membership based on new role
  • Leaver: Disable account, revoke tokens, archive data

Auditing & Monitoring

  • Logon/logoff auditing
  • Group membership change alerts
  • File access auditing on sensitive folders
  • GPO modification tracking
  • Privileged elevation logs

Outcome

This project demonstrates practical IAM engineering skills including identity governance, least‑privilege enforcement, privileged access management, lifecycle automation, and enterprise-grade security controls using Microsoft Active Directory.
